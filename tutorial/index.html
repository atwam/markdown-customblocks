    Tutorial: How to add new block types

    Setup

    For this tutorial let's use markdown from the command line. You should setup a working python environment and install both markdown and markdown-customblocks with pip install.

    $ virtualenv .venv
$ source .venv/bin/activate
$ pip install markdown markdown-customblocks

    To ensure that we have a working markdown package.

    Edit hello.md with the following content:

    ::: hello world
    **Welcome** to my home page

    Then run:

    markdown_py hello.md -f hello.html -x customblocks

    This should generate:

    <div class="hello world">
    <p><b>Welcome</b> to my home page</p>
</div>

    Since hello is not bound to any generator, the default block type container is picked.

    container generates a div with the type (hello) as class, the indented content inside, reparsed as markdown, any keyless parameter (world) as class as well, and keyword values (which we dont provide) as attributes.

    You could try to add style="background:teal" to the block headline and see what happens.

    Our first custom block

    To create our first block we need to create a python file and a config file.

    config.yml

    customblocks:
    generators:
        hello: 'hello:hello' # tag name, module, function

    hello.py 

    def hello():
    return "<h1>hello</h1>"

    To run it with the config file:

    markdown_py hello.md -f hello.html -x customblocks -c config.yml

    If you run that as is, it will complain on not finding hello module. We must tell python to look for packages also in the current directory:

    export PYTHONPATH=$PYTHONPATH:.

    If you return any string not being a single root xml fragment, you will see errors. Provoque those errors to get used to them.

    Receiving parameters

    Lets take the first parameter as the name and use an f-string to insert it capitalized into the output.

    def hello(name):
    return f"<h1>Hello {name.title()}</h1>

    Now we can do:

    :: hello peter

:: hello david

    Getting international

    def hello(name, lang='en'):
    translations = dict(
    en="Hello",
    es="Hola",
    de="Hallo",
    )   
    return f"<h1>{translations[lang]} {name.title()}</h1>

    So:

    ::: hello peter
::: hello pedro lang=es
    Renders 
    <h1>Hello Peter</h1>
<h1>Hola Pedro</h1>
    But this crashes: 
    ::: hello petro lang=it

    Making it safer:

    def hello(name, lang='en'):
    translations = dict(
    en="Hello",
    es="Hola",
    de="Hallo",
    )   
    return f"<h1>{translations.get(lang, 'Hello')} {name.title()}</h1>

    Escaping parameters and hyperscript

    Inserting parameters as verbatim is not recommended since is a source of problems. For instance:

    ::: hello :->

    We could escape parameters everywhere but this renders into a unreadable code. Instead, let's use the hyperscript tool, that transparently does just that:

    from customblocks.utils.hyperscript import E

def hello(name, lang='en'):
    translations = dict(
        en="Hello",
        de="Hallo",
        es="Hola",
    )
    return E('h1',
        translations.get('lang', 'Hello'),
        ' ',
        name.title(),
    )

    The E first parameter is the tag. Next parameters are the content.

    Excess positional parameters

    ::: hello Albert Einstein

    This will assign Albert as name and Einstein as language. We could quote it and it will work. But let's try to simplify things for users and take any positional argument as the name.

    If we want to take the excess positional parameters together, and not filling lang unless the explicit key is provided:

    from customblocks.utils.hyperscript import E

def hello(*name, lang='en'):
    translations = dict(
        en="Hello",
        de="Hallo",
        es="Hola",
    )
    return E('h1',
        translations.get('lang', 'Hello'),
        ' ',
        name.title(),
    )

    In Python functions, any parameter after the varidic * is forced to be used with key. Any positional parameter will be included in name.
